Hier ist das anspruchsvolle Multiple-Choice-Quiz zum Thema **Anforderungsanalyse (Requirements)** mit Fokus auf Arten von Anforderungen, basierend auf den Vorlesungsinhalten:

---

> [!question] **Frage 1: Welche der folgenden Aussagen beschreibt eine **funktionale Anforderung**?**
> - [ ] A) Das System muss innerhalb von 2 Sekunden auf Benutzeranfragen reagieren.
> - [ ] B) Die Benutzeroberfläche muss barrierefrei gestaltet sein.
> - [ ] C) Das System muss die Möglichkeit bieten, Mitgliedsdaten zu ändern.
> - [ ] D) Die Software muss auf Java 17 und einer MySQL-Datenbank laufen.

> [!success]- 🔑 Antwort anzeigen / Show Answer
> **Richtige Antwort: C**
>
> *Erklärung*:
> - **Funktionale Anforderungen** beschreiben **konkrete Systemfunktionen** (z. B. "Mitgliedsdaten ändern").
> - Option A und B sind **nicht-funktionale Anforderungen** (Performance bzw. Barrierefreiheit).
> - Option D ist eine **technische Anforderung** (Constraints), die weder funktional noch nicht-funktional im engeren Sinne ist, sondern eine Vorgabe für die Umsetzung darstellt.

---

> [!question] **Frage 2: Welche der folgenden Anforderungen ist eine **nicht-funktionale Anforderung**?**
> - [ ] A) Das System muss Benutzer authentifizieren können.
> - [ ] B) Die Authentifizierung muss mit OAuth 2.0 erfolgen.
> - [ ] C) Die Authentifizierung darf maximal 5 Sekunden dauern.
> - [ ] D) Die Authentifizierung muss in der Datenbank gespeichert werden.

> [!success]- 🔑 Antwort anzeigen / Show Answer
> **Richtige Antwort: C**
>
> *Erklärung*:
> - **Nicht-funktionale Anforderungen** definieren **Qualitätseigenschaften** (z. B. Performance wie "maximal 5 Sekunden").
> - Option A ist eine **funktionale Anforderung** (Systemfunktion).
> - Option B und D sind **technische Anforderungen** (Constraints zu Protokoll bzw. Speicherung).

---
> [!question] **Frage 3: Welche Aussage zu **technischen Anforderungen** ist korrekt?**
> - [ ] A) Technische Anforderungen sind immer funktional, da sie die Implementierung beschreiben.
> - [ ] B) Technische Anforderungen sind subjektiv und hängen von der Meinung des Kunden ab.
> - [ ] C) Technische Anforderungen können sowohl funktional als auch nicht-funktional sein, je nach Kontext.
> - [ ] D) Technische Anforderungen sind immer nicht-funktional, da sie keine Systemfunktionen beschreiben.

> [!success]- 🔑 Antwort anzeigen / Show Answer
> **Richtige Antwort: C**
>
> *Erklärung*:
> - **Technische Anforderungen** (z. B. "Programmiersprache Java") können **funktional** sein, wenn sie eine Systemfunktion einschränken (z. B. "Das System muss in Java implementiert werden").
> - Sie können auch **nicht-funktional** sein, wenn sie Qualitätsmerkmale betreffen (z. B. "Das System muss auf Linux laufen").
> - Option A, B und D sind zu pauschal und ignorieren die Mehrdeutigkeit technischer Anforderungen.

---
> [!question] **Frage 4: Warum sind **Metriken** bei nicht-funktionalen Anforderungen wichtig?**
> - [ ] A) Weil sie die subjektive Natur nicht-funktionaler Anforderungen messbar und bewertbar machen.
> - [ ] B) Weil sie die Entwicklungskosten reduzieren.
> - [ ] C) Weil sie die funktionalen Anforderungen ersetzen.
> - [ ] D) Weil sie die Architektur des Systems automatisch generieren.

> [!success]- 🔑 Antwort anzeigen / Show Answer
> **Richtige Antwort: A**
>
> *Erklärung*:
> - Nicht-funktionale Anforderungen (z. B. "Benutzerfreundlichkeit") sind **subjektiv**.
> - **Metriken** (z. B. "Durchschnittliche Zeit bis zur Lösung eines Problems") machen sie **objektiv messbar** und ermöglichen eine Bewertung.
> - Option B, C und D sind irrelevant oder falsch.

---
> [!question] **Frage 5: Welche der folgenden Anforderungen ist **keine** nicht-funktionale Anforderung?**
> - [ ] A) Das System muss eine Verfügbarkeit von 99,9% haben.
> - [ ] B) Das System muss in der Lage sein, Mitgliedsbeiträge automatisch zu berechnen.
> - [ ] C) Die Benutzeroberfläche muss eine Kontraststärke von mindestens 4,5:1 erfüllen.
> - [ ] D) Die Antwortzeit auf Suchanfragen darf 1 Sekunde nicht überschreiten.

> [!success]- 🔑 Antwort anzeigen / Show Answer
> **Richtige Antwort: B**
>
> *Erklärung*:
> - **Nicht-funktionale Anforderungen** beschreiben **Qualitätseigenschaften** (Verfügbarkeit, Usability, Performance).
> - Option B beschreibt eine **funktionale Anforderung** (konkrete Systemfunktion: "Mitgliedsbeiträge berechnen").
> - Option A, C und D sind nicht-funktional (Verfügbarkeit, Barrierefreiheit, Performance).
