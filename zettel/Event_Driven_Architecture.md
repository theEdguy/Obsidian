# [[Event_Driven_Architecture]]

- **Kernkonzept:** Die **Event_Driven_Architecture** (EDA) ist ein [[Architekturmuster]], bei dem der Kontrollfluss einer Anwendung durch das Auftreten von **Ereignissen (Events)** gesteuert wird. Ein Ereignis ist eine signifikante Zustandsänderung oder ein Vorkommnis (z. B. Benutzeraktion, Systemmeldung, Sensorwert), das von einem [[Publisher]] (Ereignisproduzent) erzeugt und an interessierte [[Subscriber]] (Ereigniskonsumenten) übermittelt wird. Die Kommunikation erfolgt asynchron und entkoppelt, typischerweise über einen [[Event_Bus]] oder [[Message_Broker]]. EDA baut häufig auf dem [[Observer_Pattern]] oder [[Publish_Subscribe_Pattern]] auf, geht jedoch über reine [[Entwurfsmuster]] hinaus, indem es eine systemweite Architekturperspektive einnimmt.
