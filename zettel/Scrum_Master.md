# [[Scrum Master]]

- **Kernkonzept:** Der Scrum Master ist eine zentrale Rolle im Scrum-Framework, die als Servant Leader agiert und dafür verantwortlich ist, dass das Scrum-Team die Prinzipien, Werte und Praktiken von Scrum versteht und effektiv anwendet. Er beseitigt Hindernisse, fördert die Zusammenarbeit und unterstützt das Team bei der kontinuierlichen Verbesserung.
- **Nutzen & Zweck:** Der Scrum Master existiert, um die Produktivität und Effizienz des Entwicklungsteams zu steigern, indem er sicherstellt, dass Scrum-Prozesse korrekt umgesetzt werden. Er löst Konflikte, schützt das Team vor externen Störungen und fördert eine Kultur der Selbstorganisation. Dadurch wird die Qualität der Softwareentwicklung verbessert und die Time-to-Market verkürzt.
- **Abgrenzung & Grenzen:** Der Scrum Master sollte nicht als Projektmanager oder Teamleiter missverstanden werden, da er keine disziplinarische Autorität besitzt. Er ist kein Product Owner und trifft keine inhaltlichen Entscheidungen über das Produkt. In stark hierarchischen oder nicht-agilen Umgebungen ist die Rolle weniger effektiv, da Scrum auf Selbstorganisation und iterative Prozesse angewiesen ist. Alternativen wie klassische Projektmanager eignen sich besser für starre, planungsgetriebene Projekte.
- **Beispiel / Code:** ```java
// Beispiel für eine typische Scrum Master-Aktivität: Moderation eines Daily Scrum
public class DailyScrumModeration {
    public static void main(String[] args) {
        ScrumMaster scrumMaster = new ScrumMaster();
        TeamMember[] team = {new Developer(), new Tester(), new Designer()};
        
        scrumMaster.facilitateDailyScrum(team);
        // Output: Strukturierte Beantwortung der drei Fragen:
        // 1. Was habe ich seit dem letzten Daily erreicht?
        // 2. Was plane ich bis zum nächsten Daily zu erreichen?
        // 3. Welche Hindernisse blockieren mich?
    }
}
```
