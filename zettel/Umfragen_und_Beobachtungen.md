# [[Umfragen_und_Beobachtungen]]

- **Kernkonzept:** Systematische [[Datenerhebung]] aus einer großen [[Stichprobe]] durch [[Fragebögen]] oder [[Interviews]], um [[Trends]] oder [[Meinungen]] in der [[Software-Entwicklung]] zu analysieren.
- **Nutzen & Zweck:** Ermöglicht die [[Bewertung]] von [[Methoden]], [[Werkzeuge]] oder [[Prozessen]] aus Sicht der [[Nutzer]] oder [[Entwickler]]. Grundlage für [[Deskriptive_Statistik]] und [[Explorative_Analyse]].
- **Abgrenzung & Grenzen:** Keine [[Kausalität]], da nur [[Korrelationen]] messbar sind. Unterscheidet sich von [[Kontrollierte_Experimente]] durch fehlende Kontrolle über [[Variablen]]. Nicht geeignet für [[Hypothesenvalidierung]].
- **Beispiel / Code:** ```text
// Beispiel für Umfrage-Fragebogen (geschlossene Fragen)
1. Wie oft nutzen Sie [[Test-Driven_Development]]?
   ( ) Nie ( ) Selten ( ) Manchmal ( ) Häufig ( ) Immer
2. Wie bewerten Sie die Effektivität von [[Code_Reviews]]?
   (1) Sehr ineffektiv (5) Sehr effektiv
```
